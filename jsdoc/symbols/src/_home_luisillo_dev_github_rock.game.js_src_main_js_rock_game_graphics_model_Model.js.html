  3 /**
  4  * A model is something that can be renderer. A model is composed by a {@link rock.game.graphics.model.mesh.Mesh},
  5  * that contains its geometry and a {@link rock.game.graphics.model.material.Material} that specifies how it has
  6  * to be colored (and shaded)
  7  *
  8  * Please, note that Mesh shouldn't has state as it's something 'static' that can be shared.
  9  * To add some state you should use a ModelRenderable (please, see {@link rock.game.graphics.engine.renderer.webgl.ModelRenderer},
 10  * {@link rock.game.graphics.engine.renderer.RendererContext}, and {@link rock.game.graphics.engine.renderer.ModelRenderable} class
 11  * for more info)
 12  *
 13  * A model should be agnostic about how its data must be interpreted. It's up to you assure that data
 14  * is coherent with Renderer expectations.
 15  *
 16  * @constructor
 17  * @abstract
 18  *
 19  * @author Luis Alberto Jim√©nez
 20  */
 21 rock.game.graphics.model.Model = function(type) {
 22     rock.super_(this, arguments);
 23     this.mesh = null;
 24     this.material = null;
 25 };
